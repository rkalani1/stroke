## Drafted fixes (verified locally; ready to commit)

I drafted three small high-confidence fixes during the audit, ran

### Fix 1 — align all version markers to `package.json` (P0)

Three different versions were in flight at audit time:
`package.json` `5.21.0`, the `index.html` inline `APP_VERSION =
'v5.14.78'` (line 562), and the `?v=5.20.0` cache-buster on
`app.js`/`tailwind.css` (lines 49, 50, 736). The bootloader at
`index.html:664` clears `localStorage` whenever the inline
`APP_VERSION` doesn't match the persisted one, so the lag has been
silently triggering one-shot session resets after every release that
touched only the package version.

```diff
--- a/index.html
+++ b/index.html
-    <link rel="preload" as="script" href="app.js?v=5.20.0">
-    <link rel="stylesheet" href="tailwind.css?v=5.20.0">
+    <link rel="preload" as="script" href="app.js?v=5.21.0">
+    <link rel="stylesheet" href="tailwind.css?v=5.21.0">
@@
-            const APP_VERSION = 'v5.14.78'; // Increment this with each deployment
+            const APP_VERSION = 'v5.21.0'; // Keep in lockstep with package.json "version".
@@
-    <script src="app.js?v=5.20.0" defer></script>
+    <script src="app.js?v=5.21.0" defer></script>
```

The `service-worker.js` cache bump landed:

```diff
--- a/service-worker.js
+++ b/service-worker.js
-const CACHE_NAME = 'stroke-app-v84';
+const CACHE_NAME = 'stroke-app-v85';
```

Bumping `APP_VERSION` is the *intended* trigger for the bootloader's
one-shot localStorage clean, so this is the correct shape of the fix.

### Fix 3 — gate `applyConfigData` and user quick-links to `http(s)` only (P1)

`applyConfigData` (line ~15430) and the `quickLinks` reducer (line
~15780) accept any string in `link.url` from `config.example.json`,
optional `config.local.json`, or persisted `settings.quickLinks`. A
`javascript:`, `data:`, or `vbscript:` URL would render as a clickable
button without any guard. The fix adds one helper and one filter at
each call site:

```diff
--- a/src/app.jsx (after DEID_PATTERNS)
+        // Allowlist for any URL we render, follow, or open. Restricts to http(s)
+        // so a malformed config.local.json or imported settings cannot smuggle
+        // a javascript:, data:, or vbscript: URL into a clinician-visible button.
+        const isSafeExternalUrl = (raw) => {
+          const value = String(raw || '').trim();
+          if (!value) return false;
+          try {
+            const u = new URL(value);
+            return u.protocol === 'https:' || u.protocol === 'http:';
+          } catch {
+            return false;
+          }
+        };

--- a/src/app.jsx applyConfigData()
-                .filter((link) => link.url);
+                .filter((link) => isSafeExternalUrl(link.url));

--- a/src/app.jsx quickLinks reducer
             if (!normalizedUrl || acc.seen.has(normalizedUrl)) return acc;
+            if (!isSafeExternalUrl(normalizedUrl)) return acc;
             acc.seen.add(normalizedUrl);
```

After all three fixes were drafted, `npm run test:unit` was 427/427
green and `npm run build` produced a clean 2.5 MB bundle.

## P0 — fix before next deploy

### P0-1. Mobile main-nav is invisible to QA harness because it has no `.tab-pill` class

- **Where:** `src/app.jsx` line 16450 (`hidden sm:block` desktop nav with
  `.tab-pill`) vs line 34858 (`sm:hidden` fixed-bottom mobile nav whose
  buttons do **not** carry `.tab-pill`).
- **Symptom:** `node ./scripts/qa-smoke.mjs --local-only` mobile run
  fails because `button.tab-pill:has-text("Encounter")` is `not visible`
  on a 390 × 844 viewport.
- **Why it matters:** The desktop nav is hidden on mobile (correct);
  the mobile bottom-nav is the primary tab navigation surface, and
  this is the *clinician-on-call-from-bed* path — but it has zero
  smoke coverage. A regression that broke navigation on iPhone would
  pass CI today.
- **Fix:** Update the smoke selector to also match the mobile-nav
  button (e.g. `button[role="tab"][aria-controls="tabpanel-encounter"]`).
  Don't add `.tab-pill` to the mobile bottom-nav buttons — they have a
  different visual (vertical icon-on-top), and inheriting pill styles
  would distort them.
- **Verify:** `npm run qa -- --local-only` → no `tab-nav` issue on the
  mobile run; manual on iPhone Safari confirms tab change.

### P0-2. Search dropdown intercepts the desktop header

- **Where:** `src/app.jsx`. Header search input at line 16018
  (`onFocus` calls `setSearchOpen(true)`); empty-state quick-commands
  panel at line 16161 (`searchOpen && searchContext === 'header' &&
  searchQuery.trim().length < 2`); button row holding `New Case` and
  `#settings-menu-trigger` is rendered immediately *under* the search
  input on desktop because the parent flex container (`flex-col …
  sm:items-end`) keeps a column orientation even on `sm:`.
- **Symptom:** QA-smoke desktop and tablet runs both fail with
  *"<p class='…'>Navigation</p> from <div class='relative w-full
  sm:w-auto'>… subtree intercepts pointer events"* when trying to
  click `#settings-menu-trigger`. The `<p>` is the empty-state
  command-palette category label that auto-opens on input focus.
- **Why it matters:** Clinicians who tab/click into the search input
  cannot reach the **Export PDF**, **Export JSON**, **Share**, **Recent
  Encounters**, **Dark mode**, **Reset all data** menu items without
  pressing Escape first. This is also a PDF-export-blocked-on-rounds
  workflow break.
- **Fix:** Either (a) stop auto-opening the empty-state quick-commands
  panel on focus alone (open only after the user types or presses `/`),
  or (b) tighten the empty-state panel so it doesn't extend below the
  input's own bounding box, or (c) re-flow the header on `sm:+` to put
  the buttons on the same row as the input (use `sm:flex-row`).
  Recommend (a) for least visual disruption: keep the panel as a help
  surface, but show it only on `/` or after typed input.
- **Verify:** `npm run qa -- --local-only` → no audit-runtime-error
  on desktop or tablet; manual: tab into search input on desktop, see
  no panel; press `/` or type a letter, see the panel.

### P0-3. Two `dist/` artifacts are one cache-version behind root

- **Where:** `dist/service-worker.js` (`stroke-app-v83` vs root v84/v85),
  `dist/manifest.json` (no `shortcuts` array), and `dist/index.html`
  (no Apple touch startup images, no `format-detection: telephone=no`
  meta, no PWA `safe-area-inset-top` styles).
- **Why it matters:** `dist/` is the Capacitor `webDir` per
  `capacitor.config.json`. If a wrapper build is cut from stale `dist/`,
  the App Store build will be missing iOS startup splashes, NIHSS
  auto-tel detection will turn `1.5` and `0.9` into phone-link styling,
  and the SW cache name will look identical to a previously-deployed
  PWA — confusing rollback.
- **Fix:** Run `bash scripts/sync-dist.sh` (already exists) before
  any wrapper build. Add a CI guard that fails the build when
  `dist/{index.html,manifest.json,service-worker.js}` differs from root
  on a wrapper-touching PR.
- **Verify:** `diff -q index.html dist/index.html` and the same for
  `manifest.json` and `service-worker.js` should report identical
  files. CI should fail otherwise.

## P1 — fix this sprint

### P1-1. Mobile-bottom-nav lacks tab-keyboard support

- **Where:** `src/app.jsx` line 34858. The mobile bottom nav has
  `role="tablist"` and `role="tab"` plus `aria-selected`, but no
  `aria-controls`, no left/right keyboard arrow handling, and no
  `tabIndex` roving — so Pixel/Android screen readers can't navigate
  between tabs the way they do on the desktop nav (which has all of
  this at line 16451).
- **Fix:** Mirror the desktop nav's keyboard handler onto the mobile
  surface, and add `aria-controls={`tabpanel-${tab.id}`}` (already
  present at line 34873) plus `tabIndex={isActive ? 0 : -1}`.
- **Verify:** With VoiceOver on iOS, tab between Encounter / Management
  / Trials at the bottom of the screen.

### P1-2. CDN dependencies cached without subresource integrity

- **Where:** `service-worker.js` `CDN_ASSETS`: React 18 prod,
  React-DOM 18 prod, lucide 0.563.0, html2pdf 0.10.1. Plus
  `index.html` Google Fonts `<link>` tags.
- **Why it matters:** A compromised unpkg / cdnjs edge could serve a
  modified `react-dom.production.min.js`, and the SW would happily
  cache it for every offline session. For a clinical tool that exposes
  dose calculators, this is a meaningful threat surface even if low
  probability.
- **Fix:** Either (a) self-host these assets (and add them to
  `CORE_ASSETS`), or (b) add `integrity="sha384-…"` + `crossorigin`
  to every `<script>`/`<link>` reference and update SW to refuse
  caching responses that don't match. (a) is the cleaner solution
  because the SW already special-cases CDN hosts. Bundle size impact:
  ~150 KB minified gzipped — comfortably below current 2.5 MB bundle.
- **Verify:** `script-src 'self'` CSP audit (P1-3 below) succeeds.

### P1-3. Missing Content-Security-Policy

- **Where:** `index.html` has no `<meta http-equiv="Content-Security-Policy">`
  or equivalent. GitHub Pages doesn't let you set HTTP CSP headers, but
  a meta-CSP would cover the bulk of the surface.
- **Fix:** Add a meta CSP that allows `self`, the four font/CDN hosts
  in active use, and the html2pdf CDN; block inline scripts except for
  the bootloader (which can be hashed). Pairs with P1-2 above.
- **Verify:** Open DevTools → Network → no CSP violations on a normal
  encounter walkthrough.

### P1-4. Search dropdown column-overflow on header (UX)

Tied to P0-2: even after fixing the click-intercept, the empty-state
panel covering ~40% of the header surface on first focus is jarring.
Strongly consider lazy-rendering the panel and treating the search
field as "type to search" with a discoverable `/` shortcut instead.

### P1-5. README claims vs. actual feature surface drift

- **Where:** `README.md` line 9 lists features that include "Multi-patient
  ward census" and "Clinic and wards workflows". These exist in code
  (`src/patient-store.js`, `mgmt-tabpanel-clinic`, `mgmt-tabpanel-wards`)
  but are buried under `#/management` and have no quick-link. The README
  also says "**Pure-function matcher engine** with 100% coverage of 52
  inclusion criteria + 16 exclusions" — verified by running
  `evidence:validate` (matches exactly). Counts (31 trials, 42
  citations) also match.
- **Fix:** Add a short "How to use this on rounds" section to README
  pointing clinicians to the specific deep links. Also add the
  Pulsara/transfer/discharge note generators to the README feature list
  — they're a major surface and aren't enumerated.

### P1-6. RESUME.md / SPRINT_STATUS_*.md drift

- **Where:** `RESUME.md` was written against branch
  `feature/strokeops-v6-evidence-atlas`. That branch is now `[gone]`
  (merged). The sprint status files at root (`SPRINT_STATUS.md`,
  `SPRINT_STATUS_E.md`, `SPRINT_STATUS_L.md`, `SPRINT_STATUS_PWA.md`,
  `SPRINT_STATUS_R.md`, `SPRINT_STATUS_X.md`) describe completed work
  but are still root-level scratch notes.
- **Fix:** Move sprint statuses under `docs/sprints/` and add a tiny
  `docs/sprints/INDEX.md` that links them in chronological order. Keep
  `RESUME.md` only if there's an active sprint.
- **Verify:** Repo root contains only the README + license-grade files;
  `ls` is short.

## P2 — within 30 days

### P2-1. `app.jsx` is 34,892 lines — a single monolithic component

- **Why it matters:** The file is at the edge of what an editor can
  hold in a single context window. Every Tailwind class change touches
  the same blob; bisecting clinical regressions is harder than it
  needs to be. The recently-added evidence layer (`src/evidence/*`)
  shows the right pattern.
- **Fix (incremental, no-big-bang):** Lift each `#mgmt-tabpanel-*` block
  out into its own JSX file under `src/management/`. Lift each
  calculator drawer into `src/calculators/`. Aim for ~1k-line ceilings
  per file. The existing `src/components.jsx` (696 LOC) and
  `src/teaching.jsx` are good models. This is a 2-3 sprint refactor;
  do it 1-2 panels per week, gated by snapshot tests.
- **Verify:** Bundle size unchanged (esbuild tree-shakes); 427/427
  tests still pass after each lift.

### P2-2. Bundle is 2.5 MB minified

- **Why:** Most of it is `app.jsx` content (guideline JSON inlined,
  long recommendation strings). Splash/cold-start time on a 4G phone
  in an ambulance bay can be ~1.5s after PWA install (browser
  measurement; replicate with throttling).
- **Fix:** Lazy-load the Library / Trials / Management routes via
  `React.lazy` + `Suspense`. Move the 17 guideline JSON files into a
  separate fetch (`./guidelines/<id>.json`) — they're already in
  `src/guidelines/` but appear inlined into the bundle today.
- **Verify:** Bundle size after lazy-loading should drop to <1 MB
  initial; total stays ~2.5 MB. Lighthouse PWA score on a slow 4G
  budget should improve TBT and LCP.

### P2-3. `min-h-[32px]` vs WCAG 2.2 24×24 minimum

- **Where:** `src/app.jsx` line 33356 (one button); also a few
  search-result options (~30 px tall). All other interactive surfaces
  meet the 44×44 touch target via `index.html` global rule.
- **Fix:** Bump the one outlier to `min-h-[44px]`.
- **Verify:** Repo grep for `min-h-\[(20|24|28|30|32)px\]` returns no
  UI buttons.

### P2-4. PWA shortcuts only land on three routes

- **Where:** `manifest.json` shortcuts → encounter, calculators, trials.
- **Improve:** Add a "Wards" and "Clinic" shortcut for the rounds
  workflow once that flow stabilizes.

### P2-5. Service-worker offline analytics

The current SW correctly distinguishes navigation (offline →
offline.html) from non-navigation (offline → cached or `index.html`).
Consider logging the offline-fallback events to a tiny IndexedDB queue
for later diagnostics — this is more useful than the
qa-latency-history.json, because it captures real clinician offline
behavior.

### P2-6. Dark-mode contrast spot-check

- **Where:** `src/app.jsx` has 279 occurrences of `text-slate-{400,500}`.
  In dark mode (`html.dark`), these are remapped to `#94a3b8` on a
  `#0b1220` background — a contrast ratio of ~7.5:1 (passes AA),
  but `text-slate-400` on `bg-slate-50` (light-mode) is 3.4:1 (fails
  AA for body text).
- **Fix:** Run an automated contrast pass and replace `text-slate-400`
  with `text-slate-500` for body copy (3.4:1 → 4.6:1) where it occurs
  on light backgrounds. Spot-check the search-input kbd-hint at
  `index.html` line 16071.

## Appendix C — Things that are *good* and should not be regressed

- **Calculator dose safety.** Weight caps (TNK 25 mg / 350 kg, alteplase
  90 mg / 350 kg, PCC 5000 IU). CrCl warnings on adjusted body weight
  for BMI >30. ANNEXA-I thrombotic risk flagged with an explicit
  alternate (4F-PCC 50 IU/kg). FDA-label-accurate andexanet pathway
  for apixaban vs rivaroxaban. Edoxaban explicitly flagged
  not-FDA-approved-for-andexanet.
- **PHI hygiene.** No `fetch()` ever sends user input; the only network
  surface is loading `config.example.json` / `config.local.json` and
  the html2pdf CDN.
- **Patient store.** IndexedDB-backed with localStorage fallback;
  schema-versioned; explicit de-identification (initials, MRN-last-4,
  birth-year-only).
- **Storage TTL.** 12-hour default expiry; explicit version-mismatch
  cleanup in the bootloader (this is the same mechanism Fix 1 above
  re-aligns).
- **Hash routing.** Clean fallback chain: parsed hash → `lastActiveTab`
  uiState → legacy migration → `encounter` default.
- **Disclaimers.** Visible on README, on `offline.html`, and inside the
  app shell. Not endorsed by any institution; clinicians must apply
  their own judgment.
- **Service-worker update flow.** Does not auto-skipWaiting; surfaces
  the "A new version of Stroke is ready" banner. Mid-encounter
  clinicians are not auto-reloaded.
- **Decimal-as-tel disable.** `<meta name="format-detection"
  content="telephone=no">` is the right call for a clinical app —
  prevents iOS Safari from auto-linking NIHSS scores and INRs as
  telephone numbers.
- **Evidence atlas.** 100% matcher coverage with declarative criteria;
  separate active vs completed trials; one-way Context Bridge from
  active matcher cards to literature.
- **Disciplined CI.** Five validator chains plus 427 unit tests plus
  Playwright smoke plus adaptive-latency gating, with separate
  `unit-tests` and `build-and-smoke` jobs in the freshly-updated
  `ci.yml`.

End of audit.
